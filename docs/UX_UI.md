# UX / UI

## Mapa de navegacion
- Home publica
- Explorar torneos
- Explorar comunidades
- Login
- Registro
- Dashboard
- Mi perfil
- Mis equipos
- Mis comunidades
- Mis torneos
- Mis tokens
- Detalle de torneo
- Detalle de match
- Panel administrativo
- Panel moderador
- Panel de control de tokens

## Pantallas principales
- Home con CTA claros, torneos destacados y ranking visible.
- Registro/Login con friccion baja.
- Dashboard con actividad reciente, proximas partidas, saldo de tokens y accesos rapidos.
- Perfil gamer con identidad, cuentas conectables y logros.
- Equipos con roster, roles y estado.
- Comunidad con miembros y torneos asociados.
- Torneo con overview, reglas, participantes, bracket y matches.
- Match room con estado, reporte, confirmacion y evidencias.
- Panel moderador con cola de disputas.
- Panel admin con auditoria, torneos y control interno de tokens.

## Componentes principales
- Navbar principal.
- Sidebar de dashboard.
- Cards de torneo y comunidad.
- Tablas reutilizables.
- Badges de estado.
- Formularios con validacion.
- Modal de confirmacion.
- Match result card.
- Evidence uploader preparado.
- Timeline de auditoria.

## Flujo del jugador
1. Descubrir torneo.
2. Registrarse o iniciar sesion.
3. Recibir saldo inicial de tokens internos.
4. Completar perfil.
5. Inscribirse a torneo.
6. Hacer check-in.
7. Jugar match.
8. Reportar o confirmar resultado.
9. Consultar progreso, ranking y saldo de tokens.

## Flujo del administrador
1. Crear torneo.
2. Definir fechas y operacion.
3. Publicarlo.
4. Revisar inscripciones.
5. Habilitar check-in.
6. Generar bracket.
7. Supervisar matches y disputas.
8. Verificar o asignar tokens internos segun necesidad operativa.
9. Cerrar torneo y publicar resultados.

## Flujo del moderador
1. Ver cola de disputas.
2. Abrir detalle del match.
3. Revisar evidencias.
4. Resolver caso.
5. Registrar decision.

## Flujo del administrador global
1. Revisar metricas y salud operativa.
2. Gestionar usuarios y roles.
3. Auditar torneos, matches y disputas.
4. Configurar politicas, catalogos y reglas de tokens internos.

## Diseno responsivo
- Mobile first para consulta rapida de matches y check-in.
- Breakpoints claros para tablas, brackets y paneles.
- Acciones criticas visibles sin depender de hover.
- Formularios cortos y progresivos en mobile.

## Recomendaciones UX
- Priorizar claridad competitiva sobre ruido visual.
- Mostrar estados del torneo y del match de forma muy evidente.
- Separar visualmente las acciones de jugador frente a las de administracion.
- Usar lenguaje consistente para check-in, bracket, resultado y disputa.
- Evitar ambiguedad entre tokens internos, recompensas y premios monetarios futuros.
- Incluir empty states utiles y llamados a la accion.
- Inspirarse en plataformas profesionales tipo Challengermode en estructura, densidad informativa y foco competitivo, pero con identidad propia y sin elementos de apuesta.
